## [v1.6.4] - 2026-04-26

### 주요 변경

#### 자산/가계부 개편
- 기존 자산/가계부 화면을 자산 홈과 가계부 중심 구조로 재정리했습니다.
- 자산 홈에서 생활비, 적금, 투자, 여가비 계좌와 보유 자산을 분리해 보여주도록 개선했습니다.
- 보유 자산 등록과 거래 등록 흐름을 분리해, 자산 등록에서는 예금/적금과 투자만 다루도록 정리했습니다.
- 거래 등록 시 보유 자산 기준으로 거래 자산을 선택하고, 실제 잔액이 해당 자산에 반영되도록 수정했습니다.
- 금액 입력은 1원 단위까지 허용하고, 거래 목록과 자산 상세 금액은 원 단위 그대로 표시하도록 보정했습니다.
- 월별 예산 입력을 가계부 상단 요약 영역에 직접 편집하는 방식으로 추가했습니다.

#### 캡처 채우기와 OCR
- 은행/카드 앱 거래내역 캡처를 OCR로 분석해 여러 거래를 검토 큐에 올리는 캡처 채우기 기능을 추가했습니다.
- 날짜 오른쪽의 일별 수입/지출 합계는 OCR 거래로 등록하지 않도록 제외했습니다.
- 아이콘, 승인취소, 내부 계좌이체, 같은 금액의 입금/출금 쌍을 거래 후보에서 제외하도록 보정했습니다.
- `OPENAI`, `입출금통장 이자`, 하단 거래 누락 등 캡처 파싱 정확도를 개선했습니다.
- 캡처 선택 후 버튼 라벨에 파일명이 표시되도록 수정하고, 현재 항목 등록/전체 등록/건너뛰기/취소 흐름을 정리했습니다.
- 카테고리 목록을 복구하고 드롭다운 높이만 compact하게 제한했습니다.

#### 반복 작업
- 반복 작업은 완료/미완료가 되면 기존 발생분을 과거 기록으로 남기고 다음 사이클의 새 `pending` 마스터를 생성하도록 재구성했습니다.
- `repeatGroupId`, `isRepeatMaster`, `repeatSourceId`, `occurrenceDate` 메타데이터를 보강하고 normalize 단계에서 기존 데이터도 보정합니다.
- 매일, 매주, 요일별, 매월, N일마다 반복을 할일과 일정 모두에서 처리하며, 중복 생성과 종료일 이후 생성을 방지합니다.

#### 작업/프로젝트/인박스 UI
- 프로젝트 카드는 기존 카드 목록을 유지하고, 열기 시 Inbox/할일/일정/리소스 섹션이 있는 상세 팝업을 띄우도록 정리했습니다.
- 프로젝트와 Inbox 수정 기능 및 수정 취소 흐름을 보강했습니다.
- 인박스/프로젝트 탭에 불필요한 작업 추가 폼이 남는 문제를 수정했습니다.
- 오늘 작업, 선택한 날짜 항목, 달력 카드, 요약 팝업을 더 compact하게 정리했습니다.
- 작업 카드 정렬 기준을 할일은 마감 시간, 일정은 종료 시간 기준으로 변경했습니다.
- 선택한 날짜 항목 추가 팝업의 색상 목록을 수정창과 통일하고, 선택한 색상이 실제 항목에 반영되도록 수정했습니다.

#### 모바일과 팝업 안정화
- 모바일에서 버튼이 세로로 늘어지는 문제와 팝업 크기, 반복 요일 UI, 요약 항목 UI를 조정했습니다.
- 팝업을 드래그하다가 바깥에서 놓았을 때 닫히는 문제를 방지했습니다.
- 설정 팝업과 로그인/로그아웃 흐름을 설정 안으로 정리했습니다.
- 브라우저 확장 프로그램의 비동기 메시지 오류가 앱 콘솔 오류처럼 노출되지 않도록 방어했습니다.
- label 연결 누락 경고를 줄이도록 폼 구조를 보정했습니다.

#### 캐시

## [v1.6.3] - 2026-04-26

### 수정

#### 반복 작업 마스터 승계
- 반복 작업이 완료 또는 미완료 상태가 되면 기존 발생분은 과거 기록으로 남기고 다음 사이클의 새 작업을 `pending` 마스터로 생성하도록 수정했습니다.
- `repeatGroupId`, `isRepeatMaster`, `repeatSourceId`, `occurrenceDate` 메타데이터를 보강해 반복 시리즈와 현재 활성 발생분을 안정적으로 구분합니다.
- 이미 같은 반복 그룹과 같은 발생일의 pending 마스터가 있으면 새 항목을 만들지 않도록 중복 생성을 방지했습니다.
- 매일, 매주, 요일별, 매월, N일마다 반복을 할일과 일정 모두에서 처리하도록 정리했습니다.
- 일정 반복은 시작일과 종료일의 기간을 유지한 채 다음 발생분을 생성합니다.
- 반복 종료일 이후에는 다음 발생분을 생성하지 않도록 보정했습니다.
- 예전 저장 데이터에 반복 메타데이터가 없으면 normalize 단계에서 자동 보정합니다.

## [v1.5.0] - 2026-04-24

### 주요 기능 추가

#### AI 보상 지갑
- 할일/일정 완료 시 AI가 난이도와 중요도를 판단해 코인을 지급하는 보상 시스템을 추가했습니다.
- 1코인 = 1,000원 기준으로 취미 예산을 계산하도록 대시보드 지갑을 연결했습니다.
- 보상 판단 기준을 Eisenhower 4분면 기준으로 조정해 1사분면은 높게, 4사분면은 낮게 평가하도록 개선했습니다.
- 여행 계획, 사적 약속, 비즈니스 약속, 알바/업무성 일정 등을 더 세분화해 보상 밸런스를 조정했습니다.
- 실패 상태 전환 시 보상의 일부를 차감하고, 완료 상태를 되돌리면 보상이 자동 회수되도록 했습니다.

#### 보상 UI와 로그
- 할일/일정 카드, 캘린더, 타임라인에 예상 보상 코인 배지를 표시했습니다.
- 코인 표시를 텍스트 `C` 대신 코인 아이콘 기반 UI로 변경했습니다.
- 대시보드 필터와 AI 보상 지갑을 연동해 현재 필터 범위에 맞는 보상 로그를 보여주도록 했습니다.
- 보상 로그는 현재 상태만 남도록 정리하고, 최대 5개씩 페이지네이션되도록 개선했습니다.
- 코인 사용 기록을 수정/삭제할 수 있도록 했습니다.

#### 자산과 통장 분리 관리
- 현재 자산 기능에 생활비 통장, 여유자금 통장 구분을 추가했습니다.
- AI 보상 코인 잔액을 기준으로 여유자금 목표 금액을 계산하고, 실제 여유자금 통장 금액과 비교하도록 했습니다.
- 통장 분리 관리 UI를 추가해 생활비, 여유자금, AI 보상 목표를 한 화면에서 확인할 수 있도록 했습니다.

#### OCR 입출금 개선
- 캡쳐 분석으로 여러 입출금 내역을 추가할 때 저장 후 창이 닫히지 않고 다음 항목을 이어서 검토하도록 수정했습니다.
- 사용자가 OCR 검토 창을 직접 닫거나 취소하면 남은 OCR 큐를 정리하도록 했습니다.
- OCR 거래명/가맹점명이 과거 입출금 내역과 일치하면 기존에 사용자가 입력했던 카테고리, 서브카테고리, 결제수단, 가맹점, 태그, 색상을 자동으로 불러오도록 했습니다.
- 과거 입력값을 불러와도 결제 날짜, 시간, 금액, 제목은 이번 OCR 사진 기준으로 유지하도록 했습니다.

#### PWA와 오프라인 사용성
- 웹앱 매니페스트, 앱 아이콘, 서비스 워커를 추가했습니다.
- 정적 리소스 캐시 버전을 관리해 새 JS/CSS 변경사항이 더 안정적으로 반영되도록 했습니다.
