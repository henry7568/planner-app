기존 `app.js`에 몰려 있던 기능을 역할별 파일로 분리함.

분리된 파일:
- `utils.js`
- `storage.js`
- `auth.js`
- `timePicker.js`
- `repeat.js`
- `renderItems.js`
- `plannerItems.js`
- `calendar.js`
- `dashboard.js`
- `finance.js`
- `financeOcr.js`
- `plannerUI.js`

목표:
- `app.js`를 메인 컨트롤러 역할로 축소

- 디버깅 범위 축소

### 12. 선택 날짜 팝업 기능 개선
- 선택 날짜 팝업에서 항목 추가 시 별도 popup 추가창 유지
- 기존 카드 클릭 시 수정창(`startEdit -> openEditPopup`) 흐름은 유지
- 선택 날짜용 추가창에 색상 / 태그 입력 추가
- 추가 후 선택 날짜 팝업 내용이 바로 다시 렌더되도록 수정
- 수정 / 삭제 후에도 선택 날짜 팝업이 즉시 갱신되도록 수정

---

### 13. 선택 날짜 추가창 UI 정리
- 선택 날짜 팝업 상단의 `추가` 버튼은 기본 상태에서 표시
- `추가` 버튼 클릭 시 선택 날짜용 추가창(`popupQuickAddForm`) 열림
- 선택 날짜용 추가창은 별도 popup 구조 유지
- 수정창과 선택 날짜 추가창 흐름을 분리

---

### 14. 달력 카드 스타일 조정

### 15. 안정화 및 오류 수정
수정 과정에서 발생한 주요 오류를 정리하고 해결함.
- 중복 선언 오류
- 초기화 순서 오류
- import 경로 오류
- `window.AppState` 참조 오류
- 래퍼 함수 누락 오류
- 모듈 분리 후 연결 누락 오류

대표 수정 내용:
- import 후 원본 함수 중복 삭제
- DOM 참조 선언 순서 정리
- 모듈별 설정 블록 추가
- 기존 이벤트 핸들러와 래퍼 함수 복구
- 팝업 / 수정창 / 저장 흐름 분리
